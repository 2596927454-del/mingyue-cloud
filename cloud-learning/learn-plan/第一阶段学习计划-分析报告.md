第一阶段学习计划
一台服务器，一个网站
2026年5月  |  路线图阶段：第一阶段（第1~2周）  |  难度：入门
一、问题说明
1.1 问题描述
学习者即将开始云计算的第一阶段学习，需要完成以下目标：亲手把自己写的网页放到公网上，用域名能访问。当前状态：已注册云账号，安装好 VS Code 和 Git，掌握基本的 Linux 命令（cd、ls、mkdir 等）。
1.2 学习背景与意义
第一阶段是整个云计算学习最重要的一块基石。它的核心价值不在于做出一个网页（网页本身非常简单），而在于让你亲手跑通以下链路：
•代码 → 服务器 → 公网访问：理解"部署"到底是什么意思
•域名 → DNS → IP 地址：理解互联网最基本的寻址机制
•SSH → Linux 命令行 → Nginx：第一次真正操作一台远程服务器
这个阶段会非常痛苦——你会遇到各种报错、各种连不上、各种"为什么不行"。但这是刻意设计的：每解决一个问题，你就学会一个知识点。不要跳过任何坑。
1.3 涉及的核心概念
以下是本阶段你需要理解的核心概念，按学习顺序排列：
概念	在本阶段你需要知道的程度
HTML/CSS	网页的骨架和皮肤。你只需要知道最基本的标签结构（html/head/body/div）即可
云服务器 (ECS/CVM)	云厂商租给你的一台虚拟电脑，运行在遥远的数据中心里，24小时开机
SSH	Secure Shell，加密的远程登录协议。你在自己电脑上敲命令，远程服务器执行
Linux 命令行	服务器的操作系统界面。第一阶段需要掌握约10个命令
Nginx	一个高性能的 Web 服务器软件，负责接收浏览器请求并返回网页文件
域名 (Domain)	人类能记住的网址，如 baidu.com
DNS	Domain Name System，把域名翻译成 IP 地址的"电话簿"
防火墙/安全组	云服务器的门禁系统，决定哪些端口可以被外部访问

二、解决方式
2.1 方案概述
总体思路：先本地做最简单的网页 → 启动一台云服务器 → 通过 SSH 远程连接 → 装 Nginx → 把网页放上去 → 配域名。整个流程约需 2 周，每天投入 1~2 小时。
架构图（文字版）：
你的电脑   编写 HTML →    云服务器 (ECS)   Nginx 托管网页 →    浏览器   通过域名访问网页
2.2 详细步骤（每日任务清单）
第 1~2 天：制作个人简历页
目标：用 HTML/CSS 写一个能在本地浏览器打开的自我介绍页面
1.创建项目文件夹，新建 index.html 文件
2.写入基本 HTML 结构：html、head（设置中文编码）、body
3.添加内容：姓名、学校/专业、一段自我介绍、一个你正在学云计算的说明
4.用 CSS 做简单美化：居中、字号、颜色（5~10 行 CSS 足够）
5.验收：双击 index.html 能在浏览器中看到页面
提示： 不要追求好看，不要学 CSS 框架（Bootstrap 等）。这个页面只是"部署的素材"，不是学习重点。5 分钟能写完最好，最长不要超过 1 小时。
第 3~4 天：开通云服务器 + 初识 Linux
目标：成功 SSH 连接到服务器，能在命令行中自由操作
6.登录云厂商控制台，选择"云服务器 ECS/CVM"，点击"创建实例"
7.配置选型：
•操作系统：Ubuntu 22.04（推荐）或 CentOS 7.9
•规格：最低配（1核 1GB/2GB 内存），学生免费额度完全够用
•网络：勾选"分配公网 IP"，按流量计费（最低挡）
•安全组：确保 22（SSH）和 80（HTTP）端口已放行
8.创建完成后，在控制台找到"公网 IP"地址，复制下来
9.在本地终端执行 SSH 连接：ssh root@你的公网IP
10.验收：成功登录后看到 Ubuntu 欢迎信息，能执行 ls、pwd、whoami 等命令
本阶段需要熟悉的 Linux 命令（10个）：
命令	用途
pwd	查看当前所在目录
ls	列出当前目录下的文件
cd	切换目录
mkdir	创建目录
cp / mv	复制/移动文件
rm	删除文件
vim/nano	编辑文件（至少会打开、编辑、保存、退出）
cat	查看文件内容
chmod	修改文件权限
systemctl	启动/停止/查看服务状态

第 5~6 天：安装 Nginx + 部署网页
目标：通过公网 IP 能访问到你的网页
11.在服务器上安装 Nginx：
12.Ubuntu: sudo apt update && sudo apt install nginx -y
13.CentOS: sudo yum install nginx -y
14.启动 Nginx：sudo systemctl start nginx
15.设置为开机自启：sudo systemctl enable nginx
16.用 scp 或直接 vim 把 index.html 上传到 /var/www/html/
17.验收：浏览器输入 http://你的公网IP，看到你的个人简历页
第 7 天：域名 + DNS
目标：用域名代替 IP 地址访问网站
18.在云厂商或域名注册商购买一个域名（选 .top .xyz 等最便宜的，几块钱/年）
19.在域名控制台添加一条 A 记录：主机记录填 @ 或 www，记录值填你的服务器公网 IP
20.等待 DNS 生效（通常 1~10 分钟），用 ping 你的域名 验证解析是否生效
21.验收：浏览器输入你的域名，能访问到个人简历页
2.3 核心原理分析
当你输入网址并回车，背后发生了什么？
域名 → DNS 解析出 IP → TCP 连接 :80 → 安全组检查放行 → HTTP 请求到达 Nginx  ▎  → Nginx 返回 HTML 文件 → 浏览器解析渲染成页面
22.浏览器把域名发给 DNS 服务器，DNS 返回对应的 IP 地址。浏览器拿到 IP 后，向服务器的 80 端口发起 TCP 连接（HTTP 默认端口）。安全组检查放行后，浏览器发送获取首页文件请求"。请求到达服务器的 80 端口(Nginx 进程监听端口)。Nginx会根据配置找到 /var/www/html/index.html 文件，读取内容并返回html文件，最后浏览器解析渲染成页面。
这正是本阶段要帮你建立的"全局视野"。完整流程如下：
23.DNS 解析：浏览器把域名发给 DNS 服务器，DNS 返回对应的 IP 地址。这个过程就像你查电话簿——输入名字，得到号码。DNS 是全球分布式的，所以你刚添加的解析记录不会立刻在全球生效（这就是"DNS 传播延迟"，通常 1~10 分钟）。
24.TCP 连接建立：浏览器拿到 IP 后，向服务器的 80 端口发起 TCP 连接（HTTP 默认端口）。这是浏览器和服务器之间的"电话线"。
25.HTTP 请求：连接建立后，浏览器发送"GET / HTTP/1.1"这样的请求，意思是"请把首页文件发给我"。
26.Nginx 处理：请求到达服务器的 80 端口，Nginx 进程正在监听这个端口。它根据配置找到 /var/www/html/index.html 文件，读取内容并返回。
27.安全组检查：请求到达云服务器之前，会先经过安全组（云防火墙）。如果安全组没有放行 80 端口，请求在到达 Nginx 之前就被丢弃了——这就是"防火墙端口没开"这个常见坑的本质。
28.浏览器渲染：浏览器收到 HTML 文件，解析标签结构，渲染成你看到的页面。
SSH 原理浅析
SSH 使用非对称加密：服务器持有私钥，首次连接时会把公钥发给你。之后的通信都用这对密钥加密，中间人截获了也解不开。这就是为什么 SSH 连接是"安全的 Shell"。当你用 ssh root@IP 时：
29.你的 SSH 客户端向服务器的 22 端口发起 TCP 连接
30.服务器返回它的公钥指纹（首次连接会提示你确认）
31.双方协商出一个临时对称密钥，后续通信都用这个加密
32.你输入密码或用密钥文件认证身份
33.认证通过后，你输入的命令被加密传输到服务器执行，结果加密传回
2.4 验证检查清单
完成以下每一项检查，才算真正通过了第一阶段：
检查项	通过标准
本地访问	双击 index.html，浏览器能正常显示
SSH 连接	能从本地终端 SSH 到服务器，不需要输入密码（已配置密钥登录更好）
Nginx 运行	systemctl status nginx 显示 active (running)
IP 访问	浏览器输入 http://公网IP，能看到你的简历页
域名访问	浏览器输入 http://你的域名，能看到你的简历页（确认 DNS 生效）
GitHub 仓库	代码已推送到 GitHub（包括 index.html 和你的操作笔记）
安全组	只开放 22 和 80 端口，其他端口全部关闭
基础命令	能在服务器上熟练使用 pwd、ls、cd、mkdir、cat、vim 基本操作
理解原理	能用自己的话说出"从输入网址到看到页面，发生了什么"

2.5 预计踩坑与应对
现象	可能原因	排查方法
SSH 连接超时 (Connection timed out)	安全组没有放行 22 端口；公网 IP 配错了；本地网络问题（公司/学校网络可能封了 22 端口）	1. 去云控制台检查安全组入方向是否有 22 端口规则
2. 确认用的是公网 IP 而不是内网 IP
3. 尝试用手机热点测试
SSH 连接被拒 (Connection refused)	服务器上的 SSH 服务没启动；端口号写错了	在控制台用 VNC 登录，执行 systemctl start sshd
Nginx 装不上	系统源配置问题；apt/yum 未更新	先执行 apt update 或 yum update，再安装
IP 能访问但域名不能访问	DNS 还没生效；A 记录配置错误；域名未实名认证	ping 域名看是否解析到正确 IP；等待 10 分钟再试；检查域名实名状态
能 Ping 通但浏览器打不开网页	安全组没有放行 80 端口；Nginx 没启动	检查安全组 80 端口；systemctl status nginx
Nginx 启动失败	配置文件语法错误；80 端口被占用	nginx -t 检查配置语法；netstat -tlnp 看 80 端口被谁占用
